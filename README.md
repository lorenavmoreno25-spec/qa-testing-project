Testing Funcional de SauceDemo y Validación Básica de APIs

Objetivo

Desarrollar un proyecto orientado a la práctica y aprendizaje de fundamentos relacionados con el testing funcional mediante la aplicación web SauceDemo, enfocándose en la validación de flujos y el análisis del comportamiento por medio del diseño y ejecución de casos de prueba, la identificación y documentación de incidencias en Jira con sus respectivas evidencias y la validación de escenarios positivos y negativos. De manera complementaria, se realizaron pruebas básicas de APIs utilizando Postman para practicar la validación de requests, responses y códigos de estado HTTP.

Alcance

El proyecto se desarrolló sobre la aplicación web SauceDemo e incluyó pruebas funcionales manuales orientadas a los módulos de autenticación, inventario, carrito de compras y checkout mediante el diseño y ejecución de escenarios positivos y negativos. Se contempló la documentación de incidencias con evidencias y su gestión básica en Jira. Complementariamente, se realizaron pruebas básicas de APIs en Postman mediante la ejecución y análisis de requests utilizando métodos HTTP, validación de responses y revisión de códigos de estado.

Herramientas utilizadas

- SauceDemo – Aplicación web utilizada para la ejecución de pruebas funcionales.

- Jira – Gestión y documentación básica de incidencias (bugs).

- Postman – Validación básica de APIs mediante ejecución de requests y análisis de responses.

- Excel – Diseño y organización de casos de prueba, resultados y trazabilidad.

- GitHub – Organización y publicación del proyecto y sus evidencias.

Actividades realizadas

- Diseño y ejecución de casos de prueba funcionales sobre la aplicación web SauceDemo.

- Validación de escenarios positivos y negativos relacionados con autenticación, ordenamiento de productos, carrito de compras y proceso de checkout.

- Identificación, documentación y seguimiento básico de incidencias mediante Jira con evidencias asociadas.

- Registro y organización de resultados de pruebas y trazabilidad en Excel.

- Ejecución de pruebas básicas de APIs en Postman mediante métodos HTTP, análisis de responses y validación de códigos de estado.

Resultados principales

- Diseño y ejecución de 9 casos de prueba funcionales manuales sobre SauceDemo mediante escenarios positivos y negativos.

- Identificación y documentación de 2 incidencias confirmadas y 1 observación funcional pendiente de validación contra requerimientos.

- Gestión básica de incidencias y evidencias mediante Jira.

- Organización de casos de prueba, resultados y trazabilidad en Excel.

- Ejecución de 13 requests en Postman utilizando distintos métodos HTTP (GET, POST, PUT y DELETE).

- Validación y análisis de responses, headers y códigos de estado HTTP como 200, 201, 401, 404 y 500.

- Consolidación del proyecto y sus evidencias en GitHub como portafolio de práctica QA.

Estructura del repositorio

- Test-Cases: Casos de prueba diseñados y ejecutados para SauceDemo.

- Bug-Reports: Registro y documentación de incidencias identificadas durante las pruebas.

- Evidence: Evidencias visuales organizadas por herramienta y escenario probado.

- Postman: Colección exportada y evidencias asociadas a las pruebas de APIs.

- Jira: Capturas relacionadas con la gestión y documentación de incidencias.

- README.md: Descripción general del proyecto, alcance y resultados obtenidos.
